# 图印工坊 色彩规范文档

## 1. 设计原则

本应用采用 **黑、白、蓝** 三色组合，强调工具性、专业性与信息清晰度：

- **功能优先**：颜色仅用于引导注意力、区分层级和反馈状态，不追求装饰效果。
- **高可读性**：所有文本与背景组合均满足 WCAG 2.1 AA 级可访问性标准（对比度 ≥ 4.5:1）。
- **视觉一致性**：全站 UI 组件统一使用本规范定义的颜色变量，避免硬编码色值。
- **简洁克制**：通过灰阶建立信息层级，蓝色仅作为强调色少量使用。

## 2. 色彩系统

### 2.1 主色（蓝色系）

| 名称 | 色值 | 用途 |
|------|------|------|
| Primary | `#1E88E5` | 品牌主色、图标、步骤序号背景、进度条、选中态边框 |
| Primary Dark | `#1565C0` | 主按钮背景、重要强调、hover 状态 |
| Primary Darker | `#0D47A1` | 主按钮 hover/press 背景 |
| Primary Light | `#E3F2FD` | 选中态背景、轻量提示背景 |
| Primary Lighter | `#F5FAFF` | 拖拽/悬停时的浅背景 |

### 2.2 中性色（黑白灰阶）

| 名称 | 色值 | 用途 |
|------|------|------|
| White | `#FFFFFF` | 卡片背景、按钮文字、输入框背景 |
| Gray 50 | `#F8FAFC` | 页面背景、上传区背景、画布容器背景 |
| Gray 100 | `#F1F5F9` | 备用浅背景 |
| Gray 200 | `#E2E8F0` | 边框、分隔线、卡片描边 |
| Gray 300 | `#CBD5E1` | 禁用按钮背景、拖拽区默认边框 |
| Gray 400 | `#94A3B8` | 非关键图标、占位符 |
| Gray 500 | `#64748B` | 弱化说明文字（仅用于纯白背景） |
| Gray 600 | `#475569` | 次要文本、标签、表单说明、footer 文字 |
| Gray 700 | `#334155` | 次级标题、按钮文字 |
| Gray 800 | `#1E293B` | 深色辅助元素 |
| Gray 900 | `#0F172A` | 主标题文字、hero 背景深色端 |
| Black | `#000000` | 极强调（当前未使用，保留扩展） |

### 2.3 功能色

| 名称 | 色值 | 用途 |
|------|------|------|
| Success | `#16A34A` | 成功状态（预留） |
| Error | `#DC2626` | 错误提示（预留） |

## 3. 应用场景对照

| 元素 | 前景色 | 背景色 | 对比度 | 说明 |
|------|--------|--------|--------|------|
| 页面标题 | `#0F172A` | `#FFFFFF` | 15.3:1 | 主文本，最高层级 |
| 卡片正文 | `#0F172A` | `#FFFFFF` | 15.3:1 | 主要内容 |
| 次要说明 | `#475569` | `#FFFFFF` | 7.6:1 | 表单标签、提示 |
| Footer 文字 | `#475569` | `#F8FAFC` | 7.2:1 | 底部弱化文字 |
| 主按钮文字 | `#FFFFFF` | `#1565C0` | 5.7:1 | 核心操作按钮 |
| 主按钮 Hover | `#FFFFFF` | `#0D47A1` | 8.6:1 | 按钮按下/悬停 |
| 次级按钮文字 | `#334155` | `#FFFFFF` | 10.1:1 | 上传、辅助操作 |
| 步骤序号 | `#0F172A` | `#1E88E5` | 5.7:1 | 圆形序号背景 |
| 模式按钮激活 | `#1565C0` | `#E3F2FD` | 5.0:1 | 定位/取色选中态 |
| 输入框文字 | `#0F172A` | `#FFFFFF` | 15.3:1 | 表单输入 |
| 颜色值文字 | `#0F172A` | `#F8FAFC` | 15.0:1 | HEX 颜色代码展示 |

## 4. 可访问性说明

- 所有常规文本（14px 及以上）与背景组合对比度均 ≥ 4.5:1，满足 WCAG 2.1 AA 标准。
- 主按钮使用 `#1565C0` 背景配白色文字，对比度 5.7:1，高于 4.5:1 要求。
- 避免在 `#CBD5E1` 及更浅背景上使用小于 18px 的常规文本。
- 弱化文本优先使用 `#475569`，仅在白色背景上使用；非白色背景需进一步加深。

```css
:root {
  --primary: #1E88E5;
  --primary-dark: #1565C0;
  --primary-darker: #0D47A1;
  --primary-light: #E3F2FD;
  --primary-lighter: #F5FAFF;

  --white: #FFFFFF;
  --gray-50: #F8FAFC;
  --gray-100: #F1F5F9;
  --gray-200: #E2E8F0;
  --gray-300: #CBD5E1;
  --gray-400: #94A3B8;
  --gray-500: #64748B;
  --gray-600: #475569;
  --gray-700: #334155;
  --gray-800: #1E293B;
  --gray-900: #0F172A;
  --black: #000000;

  --success: #16A34A;
  --error: #DC2626;

  /* 应用别名 */
  --bg: var(--gray-50);
  --surface: var(--white);
  --text: var(--gray-900);
  --text-secondary: var(--gray-600);
  --text-tertiary: var(--gray-500);
  --border: var(--gray-200);
}
```

## 6. 使用规范

- 所有新增组件必须优先使用 CSS 变量，禁止直接写死色值。
- 强调色仅用于：核心操作按钮、当前步骤、进度、图标、选中态边框。
- 灰阶用于构建信息层级：越深越重要，越浅越弱化。
- 白色卡片在浅灰背景上形成明确的内容分区，增强可读性。
- 阴影使用 `rgba(15, 23, 42, x)`，与灰阶系统保持一致。
